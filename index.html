 Twitter Analytics on the Cloud Fall 2015

Design, develop, deploy and optimize fully functional web-servers to handle high loads of twitter analytics requests and build a front end system for the web service.

  1. Implemented Extract Transform and Load (ETL) on a large data set (~ 1 TB) and loaded into MySQL on EC2.

  2. Designed and optimized MySQL databases to deal with scale, improve throughput and complicate queries on EC2.

  3. Developed a distributed iterative applications using the Apache Spark framework to analyze the Twitter social graph using PageRank algorithm to find the most influential users on the network on EMR.

  4. Ranked 15th in 72 teams in the final live tests of the web server.

 Uber Stream Processing Project Fall 2015

Design and implement a solution to join multiple streams of GPS data using the Samza API and transfer these GPS data streams into a stream of rider-driver matching list.

  1. Read in multiple streams of GPS data about the location of riders and drivers generated by Apache Kafka and join these streams using Apache Samza on EMR.

  2. Processed these data according to the distance between riders and drivers to generate a list of rider-driver matchings to enable a driver matching service like Uber using Apache Samza on EMR.

  3. According to the block driver numbers in history requests, generated a price factor for the current request using Apache Samza on EMR.
