Uber stream process

Design and implement a solution to join multiple streams of GPS data using the Samza API and transfer these GPS data streams into a stream of rider-driver matching list.

  • Read in multiple streams of GPS data about the location of riders and drivers generated by Apache Kafka and joined these streams using Apache Samza on EMR.
  • Processed these data according to the distance between riders and drivers to generate a list of rider-driver matching to enable a driver matching service like Uber using Apache Samza.
  • Generated price factor for the current request according to the block driver numbers in history requests using Apache Samza.
